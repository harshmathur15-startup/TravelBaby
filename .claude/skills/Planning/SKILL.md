# Planning Skill

## Workflow
1. Start in Plan Mode — analyze before coding
2. Review and edit the plan before implementation
3. Switch to Normal Mode and implement
4. Always provide a way to verify the work (tests, expected outputs)

## Steps
- Understand the full scope of the task before writing any code
- Identify affected files and dependencies
- Break the task into small, incremental steps
- Consider edge cases and potential side effects
- Define acceptance criteria upfront
