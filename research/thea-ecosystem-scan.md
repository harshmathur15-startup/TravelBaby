## Key Patterns Observed

### 1. The field has exploded but is wide, not deep
Most repos (71 of 187) are **Minimal** — just a CLAUDE.md or simple config. Only 12 repos achieve **Comprehensive** depth. The average Claude Code setup is a CLAUDE.md with some rules. Full lifecycle setups with hooks + agents + governance + testing + MCP are rare.

### 2. Skills are the dominant extension point
Skills collections dominate the ecosystem. The biggest repos by star count are skills libraries (antigravity-awesome-skills: 1,273+ skills, everything-claude-code: 91.7k stars). Most development energy goes into "what Claude can do" rather than "how Claude is governed."

### 3. Agent governance is almost absent
Out of 187 repos, fewer than 10 implement any form of:
- **Iteration caps** (only metaswarm, claude-bootstrap, our template)
- **Write scope restrictions** (only agent-guardrails-template, our template)
- **HITL gates** (only claude-bootstrap, agent-guardrails-template, our template)
- **Agent audit logging** (only everything-claude-code, our template)

Most agent systems are "let Claude run free" with no guardrails.

### 4. No one combines SaaS boilerplate + Claude Code governance
The SaaS boilerplate space (ixartz, Saas-Starter-Kit, etc.) has zero Claude Code integration. The Claude Code space has zero SaaS infrastructure. **Our template is the only repo that combines both.** This is our unique positioning.

### 5. Session lifecycle is a solved-ish problem
Multiple repos address session continuity (claude-mem, continuous-claude, mcp-memory-keeper). The pattern is converging: hooks for session start/end + CLAUDE.md for persistent context + some form of memory compression. Our approach is standard.

### 6. Multi-agent orchestration is hot but immature
Orchestration repos (ruflo, claude-squad, claude-swarm) focus on parallelism and coordination but rarely address:
- Agent contracts (what each agent owns)
- Board communication between agents
- Institutional memory (agent profiles that persist)
- Execution order dependencies

Our agent family pattern (registry + board + profiles + execution order) is unique.

### 7. Security is Trail of Bits and almost nobody else
Trail of Bits' config and skills are the gold standard for security. Most repos have no security rules, no input validation patterns, no auth patterns. Our template's 9-rule system covering security, backend, frontend, database, performance, testing, agents is unusually thorough.

### 8. Testing enforcement is emerging
TDD Guard, claude-wizard, metaswarm, and claude-bootstrap all enforce testing. But testing as a hook (not just a skill) is rare. Our PostToolUse quality gate hook is ahead of the field.

### 9. The "awesome list" meta-game is saturated
There are at least 10 "awesome-claude-*" repos competing for the same curation role. The actual unique repos they reference overlap heavily. hesreallyhim/awesome-claude-code and rohitg00/awesome-claude-code-toolkit are the most comprehensive.

### 10. Official Anthropic repos set the floor
anthropics/claude-code (55k stars), anthropics/skills (37k stars), anthropics/claude-code-action (5k stars), anthropics/claude-code-security-review (2.8k stars) establish baseline patterns. Community repos mostly extend or reconfigure these.
